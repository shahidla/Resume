Multi-Service PAYG Application: SAP BTP Kyma Runtime, Docker, Ethereum, SAP AI Business Services

Originally published on SAP Community, Aug 2021

This prototype explored a pay-as-you-go, consumption-based application where payments are handled through an Ethereum smart contract and application services run as independent containers on SAP BTP Kyma runtime.

Components

  • A React application interacts with MetaMask and uploads files for processing.
  • A Node.js service handles uploaded files and calls SAP AI Business Services.
  • A Solidity smart contract accepts payment on the Rinkeby test network.
  • Docker and Docker Compose package the services for local execution.
  • Kompose helps convert the container setup for deployment to Kyma.

Prototype goal

The work combined microservices, containerization, Kubernetes deployment, Ethereum payments, and SAP AI Business Services. It was built as a learning prototype to understand how separately scalable services could work together in a consumption-based application model.
