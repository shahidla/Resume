Event Driven Integration Using SAP Integration Suite, Solace, HANA DB, and OpenAI Validation

Originally published on SAP Community, Nov 2025

This prototype implements an event-driven pattern using SAP Integration Suite and Solace. A JSON event is published to Solace and processed by an iFlow that applies AI-based validation, routes the result, updates a HANA Cloud table, and records a structured audit log.

Architecture

Solace acts as the event broker and SAP Integration Suite acts as the consumer and processing layer. Once a message is received from the subscribed topic, the iFlow creates a trace identifier, captures the original payload, performs an AI quality check, and routes the message based on the validation result.

Two HANA Cloud tables support the process: one stores the final validated citizen record, while the second stores processing logs including payload, status, trace identifier, and error information.

Processing sequence

  • A JSON event is published to Solace.
  • SAP Integration Suite receives the subscribed event.
  • A trace id is generated and the original payload is captured.
  • OpenAI validates the payload using a structured request.
  • Valid messages update the citizen master table.
  • All messages are written to the audit log table.
