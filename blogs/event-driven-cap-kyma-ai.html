Event Driven SAP CAP on Kyma with Agentic AI and UI Auto Refresh

Originally published on SAP Community, Dec 2025

This prototype demonstrates how a CAP application can evolve into an event-driven architecture enriched with autonomous agentic AI. The objective was to show how CAP can react to external events in real time, refresh the UI automatically, and trigger an AI workflow that identifies duplicate citizens without manual user action.

The solution combines SAP BTP Kyma runtime, SAP CAP, SAP HANA Cloud, SAP Integration Suite, a Python-based agent, and OpenAI. The design uses CAP as the service layer and event broadcaster, while the UI and AI agent independently subscribe to the citizen stream.

End-to-end flow

The process starts in SAP Integration Suite. An iFlow inserts the citizen record into HANA Cloud and then calls a CAP notification endpoint. CAP broadcasts the event through a stream endpoint using Server-Sent Events. The Fiori Elements UI listens to the stream and refreshes its binding when new data arrives.

In parallel, the Python agent listens to the same stream. When it receives a CitizenCreated event, it retrieves the citizen via CAP OData, performs duplicate detection, prompts OpenAI with a structured instruction set, and writes AI suggestions back through CAP.

Agentic AI

The agent runs as a Python container in Kyma. It reacts to citizen events, retrieves the relevant business object, evaluates likely duplicate records, and writes the recommendation back to the application. To keep the result explainable, the prototype records a thinking log with search inputs, candidate lists, scores, final reasoning, and the selected action.
