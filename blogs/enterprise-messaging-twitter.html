SAP Cloud Platform Enterprise Messaging with Twitter

Originally published on SAP Community, Oct 2020

This prototype aggregated learnings across SAP Cloud Platform Enterprise Messaging, Open Connectors, SAP AI Business Services, and SAP Conversational AI. The objective was to communicate across applications by using Twitter as an event source and routing business actions through enterprise messaging.

Prototype design

  • Twitter is considered as the event source.
  • SAP Cloud Platform Open Connectors polls and receives Twitter data.
  • SAP Conversational AI determines the intent of the tweet.
  • SAP Enterprise Messaging coordinates communication across applications.
  • Node.js services handle webhooks from Open Connectors and Enterprise Messaging.
  • SAP ABAP code pulls messages, calls document extraction, and posts replies.

Scenarios

In one scenario, a tweet triggers Open Connectors, a webhook determines intent, and a message is published to an Enterprise Messaging topic. SAP then pulls the message, creates a service request, and replies to the original tweet with the outcome.
